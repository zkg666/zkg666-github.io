函数及周边的问题总结

1.函数的作用

  • 函数的封装，就是将一段功能性的代码，放到函数里方便后期使用，当使用的时候，直接调用函数就行了；
  • 基于js封装的库，以及插件中，都是以函数进行的封装
  • 函数几乎能够在咱们整个后期课程中出现

2.创建函数的方式

  • 函数声明

    function fn () {};

  • 函数表达式

    var fn = function (){};

  • 匿名函数

    (function(){})();

    注意：==这个函数只能执行一次==

3.函数参数相关的问题

  • 形参
    • 函数括号里面的变量，用来接收实参传递过来的实际参数
    • function fn ( 形参 ) { };
    • 注意：
      • 当形参个数多于实参个数，会报出undefined
      • 使用形参和实参的时候，需要保持个数一致！！
  • 实参
    • 给形参传递具体的参数，可以传递数组，字符串…
function fn (x,y,z){
    console.log(x,y,z);
}
fn(10,20);        // 10,20，undefined

4.函数的返回值以及函数的状态

  • 函数可以分为：
    • 有返回值的函数
    • 无返回值的函数
  • 如果在函数中声明了变量，保存了值，如果不加return的话，会报处undefined
  • 如果使用了return，但是后面没有一个明确的返回值，那么也是undefined
  • 如果使用了return，后续的代码将不再继续执行
  • ==补充问题==：
    • return后面还可以再去声明一个函数
  • 函数的4种状态（了解）
    • 有参数无返回值的函数
    • 有返回值无参数的函数
    • 无返回值无参数的函数
    • 有参数有返回值的函数
function fn (x,y){
    var num = x + y;
}
var f = fn(10,20);
console.log(f);        //    undefined

5. 作用域以及作用域链

  • 全局作用域

    • 在函数外部的区域就是全局作用域
    • 在全局下定义了一个变量或者是声明了一个函数，就在全局最大的对象window上添加了一个属性

  • 局部作用域

    • 在函数内部的区域就属于局部作用域

  • 作用域链：

    • 基本定义：函数内部使用一个声明并且赋值的变量，如果当前作用域内有，那么就执行，否则会沿着当前的作用域依次向上查找，这个查找的过程形成了一个链式结构，我们称之为作用域链
    • 注意：如果在全局下都没有你要使用的变量，那么直接报错

  • 补充问题：

    • 请问，函数的形参是属于全局的还是局部的？
    • 函数的形参，是属于当前函数局部作用域下的变量，外部是无法访问到的
function fn(){
      var num = 10;
    console.log(num);    //     10
}
fn();
console.log(num);    // undefined

6.js的预解析机制

  • 预解析是什么：一个js引擎去解析的时候，他会把函数声明和变量进行提升，==函数声明提升到最前面，高于变量==
  • 预解析总结：
    • 在声明函数的时候，一定要注意不要使用同一个函数名字，否则后一个函数会把前面的函数干掉重新赋值
    • 在使用变量的时候，因为预解析会把变量进行提升，但是，赋值并不会提升
    • 预解析就是一个需要注意的“大坑”，后期开发过程中，学会主动的去规避这个问题就好了
fn();
function fn(){
    console.log(1);
}
fn();
var fn = 20;
function fn(){
    console.log(2);
}
fn();

// 结果：2,2,**fn is not a function**
/* 分析：1.两个函数名字一样，后面的函数把前面的函数重新赋值了，最后调用函数的结果就是两次2
        2.var fn = 20；已经把fn函数，给重新复制成了一个number类型，最后调用函数就会报出fn is not a function，同时，后续代码将不再解析执行
*/

7.函数的this指向

  • this因为指向不确定，只有在最后调用了才能确定this的最终指向
  • ==this出现的几个主要场景：==
    • 在全局下声明的函数，this指向的就是全局最大的对象window
    • 在对象的方法中的this指向的就是最后的调用者
    • 在BOM中的定时器，this指向的也是全局对象window
    • 在DOM中的事件处理函数中，this指向的是当前的事件源
    • 在ES6的箭头函数中，由于箭头函数没有自己的this，它的this来自于外部对象的this
function fn(){
    console.log(this);
}
fn();        // 指向window

8.函数的arguments对象的基本定义以及使用场景

  • 基本的定义：
    • arguments本身也是一个对象，是一个比较特殊的对象
    • 也称之为伪数组，因为没有真正数组上的方法
    • 但是它有属于自己的方法，详细可以去MDN文档了解
  • 作用：
    • 就是在函数传入实参的时候，用于接收实参传递进来的值
  • 补充：
    • 在ES6的箭头函数中，并没有这个特殊的对象

9.创建对象的3种方式

  • 字面量方式创建对象
var obj = {
    name:'jack',
    age:18.
    sayHi:function(){
        console.log('你好，我是一个方法');
    }
}
obj.sayHi();
  • 工厂模式创建对象
function person(name,age){
    var o = new Object();
    o.name = name;
    o.age = age;
    o.sayHi = function(){
        console.log('你好，我是一个方法');
    }
    return o;
}
var obj = person('jack',18);
obj.sayHi();
console.log(obj.name);
  • 自定义构造函数创建对象
function Person(name,age){
    this.name = name;
    this.age = age;
    this.sayHi = function (){
         console.log('你好，我是一个方法');
    }
}
var obj = new Person('jack',18);
obj.sayHi();
console.log(obj.name);
  • 现在掌握创建对象的基本方式就好

10.new干了几件事

  • 四件事
    • 创建了一个新的空对象
    • 并且把this指向了新的空对象
    • 把属性和方法给到这个新的空对象上
    • 返回这个新的对象
